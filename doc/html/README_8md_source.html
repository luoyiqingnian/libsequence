89 ### Simplest installation instructions
90 
91 ~~~
92 ./configure
93 make
94 sudo make install
95 ~~~
96 
97 The build conditions can be adjusted via the usual environment variables. To compile an optimized "release" build:
98 
99 ~~~
100 ./configure CXXFLAGS="-O3 -DNDEBUG"
101 ~~~
102 
103 To compile a debugger-friendly build:
104 
105 ~~~
106 ./configure CXXFLAGS="-O0 -g"
107 ~~~
108 
109 To change the compiler, set the C and C++ compiler variables:
110 
111 ~~~
112 ./configure CC=gcc CXX=g++
113 ~~~

115 #### Compiling unit tests and examples
116 
117 To compile unit testing suite and example programs
118 
119 ~~~
120 make check
121 ~~~
122 
123 or
124 
125 ~~~
126 cd test
127 make check
128 ~~~
129 
130 Note that the library must be built prior to "make check", but _you do not have to install the library prior ot "make check"_. The examples and unit tests are statically-linked to the version of the library that will be found in src/.libs after a "make" command. I do this so that one can perform unit tests without having to install the library. I use static linking here to avoid any possible confusion with an existing libsequence installation.
131 
132 #### Running the unit tests
133 
134 ~~~
135 cd test && sh runTests.sh
136 ~~~

138 ### More complex installation scenarios
139 
140 Some users may not have the dependent libraries installed in the standard locations on their systems. Note that "standard" means wherever the compiler system looks for header files during compilation and libraries during linking. This scenario is common on OS X systems where users have used some sort of "system" to install various libraries rather than installing from source directly. In order to accomodate such situations, the user must provide the correct path to the include and lib directories. For example, assume that the dependend libraries are in /opt on your system. You would install libsequence as follows:
141 
142 CPPFLAGS=-I/opt/include LDFLAGS="$LDFLAGS -l/opt/lib" ./configure
143 
144 make
145 
146 ~~~
147 sudo make install
148 ~~~
149 
150 Note that the modification of LDFLAGS prepends the current value of LDFLAGS if it exists. This allows for scenarios where the system's search path for libraries may have been modified by the user or sysadmin via a modification of that shell variable. (One could also do the same with CPPFLAGS, FYI.)
151 
152 ### Installing libsequence locally
153 
154 If you do not have permission to "sudo make install", you can install the library in your $HOME:
155 
156 ./configure --prefix=$HOME
157 
158 Then, when compiling any program using libsequence, you need to add
159 
160 ~~~
161 -I$HOME/include
162 ~~~
163 to any compilation commands and
164 
165 ~~~
166 -L$HOME/lib -Wl,-rpath,$HOME/lib
167 ~~~
168 
169 to any linking commands.
170 
171 When running programs linking to any of the above run-time libraries, and depending on your system, you may also need to adjust variables like LD_LIBRARY_PATH to prepend $HOME/lib to them, etc., but you'll need to figure that out on case-by-case basis, as different systems can behave quite differently.

183 ## Using libsequence to compile other programs
184 
185 If libsequence is not installed in a standard path, then you must provide the appropriate include (-I) and link path (-L) commands to your compiler. This may be done in various ways, e.g., via a configure script or your own Makefile.
186 
187 A program that depends on libsequence must provide at least the following libraries to the linker:
188 
189 ~~~
190 -lsequence -lz
191 ~~~
